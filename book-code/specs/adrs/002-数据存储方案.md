# ADR-002：使用 SQLite 存储日报历史

## 状态

已采纳

## 背景

日报生成器需要存储日报历史记录，供后续查询和审计使用。proposal.md 要求"数据存储：本地 SQLite"，但需要做一次正式评估确认此方案是否合适。

## 选项

| 选项 | 优点 | 缺点 |
|------|------|------|
| SQLite | 零部署、零运维、Python 内置支持、文件级备份 | 不支持高并发写入、数据量大时查询性能下降 |
| PostgreSQL | 功能完整、高并发、全文搜索 | 需要独立部署和运维、对 5 人团队日报杀鸡用牛刀 |
| JSON 文件 | 最简单、无需任何依赖 | 无查询能力、并发写入不安全、数据量增长后不可管理 |

## 决策

使用 SQLite

## 理由

1. proposal.md 明确要求"运行方式：定时任务，不需要常驻服务"——没有并发写入场景（每天只执行一次）
2. 5 人团队一年的日报数据约 1800 条记录，SQLite 绰绰有余
3. 零运维符合项目定位——不想为了日报工具维护一个数据库服务
4. Python 标准库 sqlite3 开箱即用，无需额外依赖
