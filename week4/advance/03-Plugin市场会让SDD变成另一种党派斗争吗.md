# Plugin 市场会让 SDD 变成另一种"党派斗争"吗

Superpowers 进 Anthropic 官方 Plugin 市场是 2026 年 Q1 SDD 生态最大的事件。149K 星不是凭空来的——一个工具被官方背书之后，会形成马太效应，越多人用越多人推荐越多人用。

学员看到这个有点担心："是不是以后大家都得用 Superpowers 才行？OpenSpec 和 BMAD 会被边缘化吗？"

我的判断是：**会出现派系倾向，但不会变成你想象的那种排他斗争**。

派系倾向是真实的。Plugin 市场降低了"装一个工具"的成本——不用查文档、不用选版本、不用配 marketplace。这种摩擦减少会让 Superpowers 成为新人的默认选择。"反正一行命令装好"，谁还去研究 OpenSpec 的七命令呢？

但排他斗争不会发生，原因有三个。

第一，**SDD 工具不互相替代**。前面那篇文章讲了，grill-me / OpenSpec / BMAD / Superpowers 是不同层级，解决不同问题。Superpowers 装得再普及，也替代不了 grill-me 的 6 行简洁，也替代不了 OpenSpec 的 change 生命周期管理。市场会扩大不会收敛。

第二，**Anthropic 官方市场不是排他渠道**。Plugin 市场更像 npm，不是 App Store。任何人可以发布自己的 plugin（OpenSpec 也可以做成 plugin 上架），用户可以同时装多个。这和 iOS/Android 那种"只能用一个"的生态完全不同。

第三，**SDD 的核心价值不在工具**。你要解决的是"先写规范再写代码"这个心智，工具是辅助。哪个工具帮你建立这个心智、让你愿意每天坚持，那个工具就是好工具。这件事没有"行业标准"答案——团队规模不一样、业务节奏不一样、个人偏好不一样，结论必然不一样。

不过有一个变化值得注意：**新人入坑路径会变窄**。两年前一个学员问"我该用什么 SDD 工具"，回答会是"先看你团队规模……"，是个分支题。两年后大概率回答会变成"先装 Superpowers"，因为它在那里、装起来零成本、社区帖子最多。这不是工具变好了，是新人选择路径被收窄了。

收窄路径不是坏事——降低入门门槛是行业成熟的标志。但作为深度学员，你应该有意识地不被"默认路径"完全主导，至少了解一下 OpenSpec 和 BMAD 在解决什么 Superpowers 解决不了的问题。这个 repo 让你装了四层工具就是这个用意——不是要你都装着用，是要你**知道选择存在**。

知道选择存在的人，和只看到一条路的人，是两种工程师。
