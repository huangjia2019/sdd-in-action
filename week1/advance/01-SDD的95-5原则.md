# SDD 的 95/5 原则

SDD 的价值分布是这样的：

- **95% 价值** 来自 "你愿意写 spec 这个行为本身" ——
  那个 "我先想清楚再让 AI 做" 的心态转变。

- **5% 价值** 来自 "用什么工具写 spec" ——
  OpenSpec / Spec-Kit / Superpowers / BMAD 哪个都行。

---

## 为什么大多数学员搞反了

大多数学员反过来了——

- 花 2 周研究 spec-kit / OpenSpec / Superpowers 哪个好
- 最后决定一个，装好
- 跑了 3 次觉得麻烦就放弃
- 回到 Vibe Coding
- 下次又重新研究另一个工具

**真正该花时间的是那 95%**——培养"写 spec 的习惯"。

---

## 怎么培养

今晚就能做：

1. 打开任何编辑器
2. 新建 `specs/<明天要做的功能>.md`
3. 写四个 H2：
   ```
   ## 要做什么
   ## 不做什么
   ## 边界 & 验收
   ## 怎么验证
   ```
4. 每个 H2 下面填 3-5 条
5. 明天写代码前先把这份 spec 贴给 AI

**不管你装不装工具 —— 这就是 SDD 的 95% 价值**。

---

不要在 5% 的事情上花 95% 的时间。工具是加速器 · 思维是引擎 · 先把引擎点着 · 再挑加速器。
