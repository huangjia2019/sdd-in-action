# AGENTS.md · 编码规范段 · 参考版

> Week 1 · 第 2 节 · B 路产出
> 把这段合并进你项目根目录的 `AGENTS.md` · 替换原有的 `## 编码规范` 段

```markdown
## 编码规范

详见 [specs/coding-standards.md](./specs/coding-standards.md)。本段仅列关键约束供 AI 执行参考。

### Python
- 格式化：black 100 行宽
- 静态检查：ruff + mypy
- docstring：Google 风格 · 公开函数必填

### TypeScript
- tsconfig：strict + noUncheckedIndexedAccess
- 格式化：prettier 100 行宽
- 文档：TSDoc · @param/@returns 必填

### 禁止项
- 魔法字符串（例外：单次用户消息）
- TODO / FIXME / XXX / HACK 提 main（只允许 NOTE）
- 任何 AI 生成的代码未经人工 review 直接 merge

### AI 生成标识
- 代码：`# Generated by <agent-name> on <date>` 注释
- commit：`[AI]` 前缀
- 所有 AI 修改文件必须在 PR description 列出

### 覆盖率
- 业务层 ≥ 90% · 工具层 ≥ 80% · 适配层 ≥ 50%
- CI 每 PR 跑 `make lint` + `make test-coverage`
```
